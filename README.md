
# Лемана Про — юнит-экономика FBS/FBO

Репозиторий переведен на отдельные CSV-тарифы и единый `app.py`.

## Что внутри

- `app.py` — основной Streamlit-калькулятор
- `data/commissions_fbs_fbo.csv` — комиссии FBS/FBO
- `data/last_mile.csv` — последняя миля
- `data/return_last_mile.csv` — возврат последней мили
- `data/zero_mile.csv` — доставка до СЦ
- `data/return_zero_mile.csv` — возврат доставки до СЦ
- `data/zones.csv` — справочник регионов и зон
- `update_tariffs_from_excel.py` — обновление CSV из новых Excel
- `requirements.txt`

## Установка

```bash
pip install -r requirements.txt
streamlit run app.py
```

## Как пользоваться

1. Запустите `streamlit run app.py`.
2. Скачайте шаблон импорта.
3. Заполните файл с товарами.
4. Загрузите файл в блоке "Каталог товаров".
5. Сохраните каталог.
6. При необходимости поправьте комиссию прямо в таблице.
7. Нажмите "Рассчитать юнит-экономику".

## Какие поля нужны в импорте

Желательные колонки:

- `SKU`
- `Название`
- `Шаблон товара`
- `Тип товара`
- `Подкатегория товара`
- `Категория`
- `Длина`
- `Ширина`
- `Высота`
- `Вес`
- `Себестоимость`
- `Текущая цена`
- `Цена акции`

Чем точнее заполнены поля `Шаблон товара / Тип товара / Подкатегория товара / Категория`, тем надежнее автоматическое определение комиссии.

## Что изменено в математике

### FBS
- склад зафиксирован как `Москва / МО`, `origin zone = 9`;
- логистика считается по объемному брейку в литрах;
- учитываются:
  - нулевая миля,
  - последняя миля,
  - невыкуп,
  - возврат после выкупа.

### FBO
- входящая логистика задается вручную на 1 единицу;
- последняя миля и возвраты считаются по тарифным таблицам;
- возврат до СЦ в FBO не добавляется.

### Финансы
- считается ожидаемая выручка, а не просто цена на карточке;
- учитывается доля продаж по акции;
- можно заложить скидку площадки;
- рекомендованная цена ищется под целевую маржу после налога.

## Как обновлять тарифы

Когда Лемана Про меняет комиссии или логистику:

1. Положите новые Excel в корень проекта.
2. Назовите их:
   - `source_commissions.xlsx`
   - `source_logistics.xlsx`

Допускается и один файл, если в нем есть нужные листы.

3. Запустите:

```bash
python update_tariffs_from_excel.py
```

4. Скрипт перезапишет CSV в папке `data/`.

## Когда код править не нужно

Код не трогаем, если изменились только:
- цифры комиссий;
- цифры тарифов;
- список регионов и зон.

В этом случае достаточно:
- заменить исходные Excel,
- запустить `update_tariffs_from_excel.py`.

## Когда код уже нужно править

Код нужно смотреть, если:
- изменились названия листов;
- изменились названия колонок;
- Лемана Про поменяла саму логику тарификации.

## Рекомендация по git

В git лучше хранить:
- `app.py`
- `data/*.csv`
- `update_tariffs_from_excel.py`

Исходные Excel можно не коммитить.
